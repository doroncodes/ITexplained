The Big PictureOOP Fundamentals

OOP stands for Object-Oriented Programming. Instead of writing one long list of instructions, you model your program as a collection of objects — things that have data and can do stuff.

Think of it like this: a Dog object has data (name, breed, age) and can do things (bark, fetch, eat). OOP lets you organise your code the same way.

📐 The 4 Pillars of OOP

1. Encapsulation

Bundle data and methods together. Hide the fields (make them private) so only the class itself can change them directly.

🔒 Like a capsule that keeps medicine protected.

2. Abstraction

Show only what is necessary. Hide the complex details — the user just calls a method without needing to know how it works inside.

🎮 Like a TV remote — you press Power, you don't need to know the electronics.

3. Inheritance

A child class inherits fields and methods from a parent class. Write shared code once, reuse it everywhere.

👪 Like how children inherit traits from their parents.

4. Polymorphism

The same method name can behave differently depending on the object it's called on.

🐾 A Dog and a Cat both have speak() but one barks and one meows.

⭐
IEB Exam: always apply encapsulation Make all fields private. Provide public getters and setters. If you forget this, you lose marks for every question that has a class.

How to Write a Class — Step by Step

1

Declare the class

Write public class ClassName { — the name must start with a capital letter and match the filename.

2

Add private fields

These are the variables that store data for each object. Always make them private.

3

Write the constructor

Same name as the class, no return type. Uses this.field = parameter to set each field.

4

Write getters (and setters if needed)

A getter returns a field value. A setter lets outside code safely change a field (with validation if needed).

5

Add other methods

These are the behaviours — things the object can do. E.g. accelerate(), toString().

Full class + creating objectsJAVA
public class Car {

    // ① Fields — always private
    private String brand;
    private int    year;
    private double speed;

    // ② Constructor — same name, no return type
    public Car(String brand, int year) {
        this.brand = brand;   // this.brand = field, brand = parameter
        this.year  = year;
        this.speed = 0;
    }

    // ③ Getter
    public String getBrand() { return brand; }
    public double getSpeed() { return speed; }

    // ④ Method
    public void accelerate(double amount) {
        speed += amount;
    }

    public String toString() {
        return brand + " (" + year + ") — speed: " + speed;
    }
}

// ─── In main() ─────────────────────────────────────────────
Car myCar = new Car("Toyota", 2022);   // creates an object
myCar.accelerate(80);
System.out.println(myCar.toString()); // Toyota (2022) — speed: 80.0

Parent & Child ClassesInheritance

Inheritance means a child class automatically gets all the fields and methods of a parent class. You use the keyword extends. This prevents code duplication.

🌳 Inheritance Tree — Animal example
Animal
name, age · eat() · sleep()
Dog
breed · fetch() · bark()
GoldenRetriever
coatColour
Labrador
guideStatus
Cat
indoor · purr() · scratch()

Every Dog IS-AN Animal. GoldenRetriever IS-A Dog IS-AN Animal. Each level adds more detail.

How Inheritance Works — Step by Step

1

Write the Parent class normally

Put shared fields and methods here. E.g. all animals have a name and can eat.

2

Write the Child class with extends

public class Dog extends Animal — Dog now has everything Animal has.

3

Add Dog-only fields

E.g. private String breed — only dogs have a breed, not all animals.

4

Call super() first in the child constructor

super(name, age) calls the Animal constructor to set those fields. Must be line 1.

5

Add Dog-only methods (and override if needed)

E.g. fetch() is a Dog-only method. You can also override toString() from Animal.

Parent class → Child class → UsageJAVA
// ─── PARENT ────────────────────────────────────────────────
public class Animal {
    private String name;
    private int    age;

    public Animal(String name, int age) {
        this.name = name;
        this.age  = age;
    }

    public String getName() { return name; }
    public int    getAge()  { return age;  }

    public void eat() { System.out.println(name + " is eating"); }

    public String toString() { return name + ", age " + age; }
}

// ─── CHILD ─────────────────────────────────────────────────
public class Dog extends Animal {        // ← "extends" = inheritance
    private String breed;               // Dog-only field

    public Dog(String name, int age, String breed) {
        super(name, age);               // ← MUST be first line!
        this.breed = breed;
    }

    public String getBreed() { return breed; }

    public void fetch() {              // Dog-only method
        System.out.println(getName() + " fetches!");
    }

    @Override
    public String toString() {        // override the parent version
        return super.toString() + ", breed: " + breed;
    }
}

// ─── USAGE ─────────────────────────────────────────────────
Dog d = new Dog("Rex", 3, "Labrador");
d.eat();                         // inherited from Animal → "Rex is eating"
d.fetch();                       // Dog's own method      → "Rex fetches!"
System.out.println(d.toString()); // → "Rex, age 3, breed: Labrador"

Abstract Classes

An abstract class is a parent class that cannot be created directly. It's just a template. Abstract methods have no body — the child must fill them in.

Abstract: Shape
abstract area()
abstract perimeter()
← no body, subclass must implement
→
Circle extends Shape
area() { π×r² }
perimeter() { 2πr }
← provides the body
→
Rectangle extends Shape
area() { w×h }
perimeter() { 2(w+h) }
← provides the body
⚠️
Don't confuse: Overriding = child replaces a parent method (same name, same parameters). Overloading = same method name in same class, but different parameters.

Common Array Algorithms

Sum, Max, Min, Count, ReverseJAVA
int[] arr = {10, 40, 25, 60, 15};

// ① Sum all elements
int sum = 0;
for (int i = 0; i < arr.length; i++) {
    sum += arr[i];
}  // sum = 150

// ② Find the largest value
int max = arr[0];                       // start with index 0
for (int i = 1; i < arr.length; i++) { // check the rest
    if (arr[i] > max) max = arr[i];
}  // max = 60

// ③ Count how many are above 20
int count = 0;
for (int v : arr) {     // for-each (read-only)
    if (v > 20) count++;
}  // count = 3  (40, 25, 60)

// ④ Average
double avg = (double) sum / arr.length;  // cast so we get decimals

Creating the array does NOT create the objects! Student[] roster = new Student[3] gives you 3 null slots. You must fill each with roster[0] = new Student(...).
Full example — array of Student objectsJAVA
// ─── Step 1: Create the array (empty slots) ───────────────
Student[] roster = new Student[3];

// ─── Step 2: Fill each slot with an object ────────────────
roster[0] = new Student("Alice",   92);
roster[1] = new Student("Bob",     78);
roster[2] = new Student("Charlie", 85);

// ─── Step 3: Access methods on array elements ─────────────
System.out.println(roster[0].getName());   // → Alice
System.out.println(roster[2].getGrade());  // → 85

// ─── Step 4: Loop through all objects ─────────────────────
Student topStudent = roster[0];
for (Student s : roster) {
    System.out.println(s.getName() + ": " + s.getGrade());
    if (s.getGrade() > topStudent.getGrade()) {
        topStudent = s;
    }
}
System.out.println("Top: " + topStudent.getName()); // → Alice

All 4 loop types with examplesJAVA
// ── 1. for loop (you know how many times) ─────────────────
for (int i = 1; i <= 5; i++) {
    System.out.println("i = " + i);     // prints 1, 2, 3, 4, 5
}

// ── 2. while loop (don't know how many times) ─────────────
int x = 1;
while (x <= 100) {
    x = x * 2;                           // 1 → 2 → 4 → 8 → 16 → 32 → 64 → 128
}
System.out.println(x);                   // → 128

// ── 3. do-while (must run at least once — great for menus)
int choice;
do {
    System.out.println("1. Add  2. Remove  3. Quit");
    choice = sc.nextInt();
} while (choice != 3);             // note the semicolon here!

// ── 4. for-each (read every item in array, nice and clean)
int[] marks = {70, 85, 60, 95};
for (int mark : marks) {             // reads "for each mark in marks"
    System.out.println(mark);
}

// ── break (exit loop early) ────────────────────────────────
for (int i = 1; i <= 10; i++) {
    if (i == 5) break;               // stops at 5, prints 1 2 3 4
    System.out.print(i + " ");
}

// ── continue (skip one iteration) ─────────────────────────
for (int i = 1; i <= 5; i++) {
    if (i == 3) continue;            // skips 3, prints 1 2 4 5
    System.out.print(i + " ");
}

Multi-branch decisionsSwitch Statements

A switch is like a cleaner version of many if-else if statements. Use it when you're testing one variable against many exact values.

📊 switch Flowchart — switch(day)
⚠️
Don't forget break! Without it, Java keeps running into the next case. This is called "fall-through" and is usually a bug.
switch examples — int, String, fall-throughJAVA
// ── switch on int ──────────────────────────────────────────
int day = 3;
switch (day) {
    case 1:  System.out.println("Monday");    break;
    case 2:  System.out.println("Tuesday");   break;
    case 3:  System.out.println("Wednesday"); break;
    default: System.out.println("Invalid");   break;
}
// → Wednesday

// ── switch on String ───────────────────────────────────────
String season = "Winter";
switch (season) {
    case "Summer":  System.out.println("Hot!");   break;
    case "Winter":  System.out.println("Cold!");  break;
    case "Spring":
    case "Autumn":  System.out.println("Mild");   break;
    default:        System.out.println("Unknown"); break;
}
// → Cold!

// ── Useful trick: grade boundaries ────────────────────────
char grade = 'B';
switch (grade) {
    case 'A': System.out.println("Excellent!"); break;
    case 'B': System.out.println("Good!");      break;
    case 'C': System.out.println("Average");   break;
    default: System.out.println("See teacher"); break;
}
// → Good!

⚠️ The #1 String Mistake

❌ WRONG — never do this
String a = "hello";
String b = "hello";

if (a == b) {       // ← WRONG!
    // == compares memory
    // addresses, not text
}
✅ CORRECT — always use .equals()
String a = "hello";
String b = "hello";

if (a.equals(b)) {  // ← CORRECT
    // compares the actual
    // text content
}

Before your examQuick Cheat Sheet
✅ 10 Things to Always Remember in the Exam
1

Fields must always be private

Apply encapsulation — use getters and setters to access them.

2

Constructor = same name as class, no return type

Called with new ClassName(). Sets the fields using this.field = param.

3

super() must be the first line of a child constructor

Always required in inheritance when the parent has a constructor.

4

Use .equals() for Strings, never ==

== compares memory addresses. .equals() compares the actual text.

5

Array index starts at 0, last index = length − 1

An array of 5 elements has indices 0, 1, 2, 3, 4.

6

Create each object in an array of objects separately

new Student[3] gives 3 null slots. You must fill each with new Student(...).

7

Don't forget break in every switch case

Without break, Java falls through to the next case — usually a bug.

8

Random number formula: (int)(Math.random() * range) + min

Dice = (int)(Math.random() * 6) + 1. Range = max − min + 1.

9

substring(s, e): s is included, e is NOT

"Hello".substring(1, 4) gives "ell" — stops just before index 4.

10

Cast to double when dividing for a decimal result

10 / 3 = 3 (int division). (double)10 / 3 = 3.333 (correct).
